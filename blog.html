Сегодня в руки попал старый IBM PC/XT 5160. Состояние — как после ядерной
войны. Пыль, окислы, вздувшиеся конденсаторы. Но сердце еще бьется!

Первым делом:
  $ dismount /all
  $ clean --deep --isopropyl-99
  
Материнская плата выглядела как археологическая находка. Intel 8088 
на месте, но половина микросхем в сокетах окислилась. 

После чистки контактов ластиком и замены конденсаторов:
  > BOOT SEQUENCE INITIATED...
  > MEMORY TEST: 640KB OK
  > KEYBOARD... OK
  > FDD CONTROLLER... OK
  
Жив, проклятый! DOS 3.3 загрузилась с первого раза. 
Ностальгия level 9000.

Коллекция магических заклинаний для терминала:

1. Найти и удалить старые логи:
   $ find /var/log -name "*.log" -mtime +30 -exec rm {} \;
   
2. Мониторинг в реальном времени:
   $ watch -n 1 'ps aux | sort -nrk 3,3 | head -n 5'
   
3. Быстрый бэкап с прогрессом:
   $ rsync -avz --progress /source/ /destination/
   
4. Массовое переименование:
   $ for f in *.txt; do mv "$f" "${f%.txt}.bak"; done
   
5. История команд с датой:
   $ export HISTTIMEFORMAT="%d/%m/%y %T "
   
Помните: с великой силой приходит великая ответственность.
Всегда тестируйте на кошках... то есть, на тестовом сервере!

CLICK-CLACK-CLICK-CLACK...

Этот звук теперь саундтрек моей жизни. IBM Model M, 1987 года выпуска.
Buckling spring switches. Вес — как у гантели. Можно убить человека.

Коллеги сначала косились. Потом привыкли. Теперь сами хотят такую же.

Преимущества:
  - Тактильная отдача как у пишущей машинки
  - Невозможно нажать случайную клавишу
  - Служит вечно (этой клаве 37 лет!)
  - Статус в офисе +100

Недостатки:
  - Соседи слышат когда я кодирую
  - Кот боится
  - Нельзя взять в кофейню (слишком громко)
  
Но когда пишешь код на такой клавиатуре, чувствуешь себя
настоящим хакером из 80-х. Worth it!

Пора расставить точки над i.

VIM:
  + Есть везде, даже в тостере
  + Быстрый как молния
  + Modal editing — гениально
  - Кривая обучения как у ракетостроения
  - :wq или :q! — вечная дилемма

EMACS:
  + Можно сварить кофе не выходя из редактора
  + Lisp везде (для кого-то минус)
  + Org-mode — убийца всех todo-приложений
  - Нужна третья рука для комбинаций клавиш
  - Загружается дольше чем Windows 95

Мой выбор? Vim для быстрых правок, Emacs для больших проектов.
А для конфигов — nano. Да, я сказал это!

P.S. VS Code — для тех, кто не может выйти из Vim.
